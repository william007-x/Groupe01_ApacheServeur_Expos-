# Déploiement et sécurisation d’un serveur Apache sous Linux

## Présentation
Ce projet consiste à installer, configurer et sécuriser un serveur web **Apache HTTP Server** dans un environnement **Linux**. Il a été réalisé dans le cadre du cours de **Système d’Exploitation I & II** et met en pratique les bases de l’administration système et des services web.

## Objectifs du projet
- Installer et rendre opérationnel Apache HTTP Server  
- Comprendre l’architecture et le rôle des fichiers de configuration  
- Configurer des Virtual Hosts  
- Gérer correctement les permissions sous Linux  
- Mettre en place la sécurisation HTTPS (TLS)

## Environnement technique
- Système d’exploitation : Linux (Ubuntu/Debian)
- Serveur web : Apache HTTP Server
- Sécurité : OpenSSL (certificat TLS auto-signé)
- Outils : terminal Linux, commandes système

## Démarche suivie
Le projet a été réalisé selon une approche progressive :
1. Installation du serveur Apache
2. Analyse de l’architecture et des fichiers de configuration
3. Configuration des ports et des Virtual Hosts
4. Gestion des permissions et des propriétaires
5. Activation du SSL et configuration HTTPS
6. Tests de fonctionnement et validation

## Tests effectués
- Vérification de l’état du service Apache
- Accès au site via HTTP et HTTPS
- Validation de la configuration Apache
- Vérification des ports 80 et 443

## Résultats obtenus
- Serveur Apache fonctionnel
- Sites accessibles via Virtual Hosts
- Communication sécurisée par TLS
- Configuration stable et conforme aux bonnes pratiques
